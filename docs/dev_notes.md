# Dev notes

## Кнопка "Рецензия Big-O блока" в Monaco

Ручной чек-лист после изменений в `monaco_widget.py` / `bigo_controller.py` /
`bigo_bridge.py`:

1. Открыть приложение (`python main.py`).
2. Открыть папку проекта (Файл → Открыть папку).
3. Нажать кнопку Big-O анализа на верхней панели.
4. Дождаться завершения анализа: справа появится рецензия и блок
   "Источники оценок". В редакторе — подсветка блоков и Big-O лейблы.
5. Над каждым подсвеченным блоком должна быть строка view zone с подписью
   сложности, например `[O(n^2)]`, и кнопкой рецензии справа от подписи.
   Кнопка использует иконку `icons/ai_ricense_block.png`.
6. Сузить окно по X и проверить, что кнопка остаётся рядом с подписью
   `[O(...)]`, а не уезжает в конец первой строки блока.
7. Кликнуть по кнопке:
   - в правой панели вместо рецензии проекта появляется заголовок
     "Рецензия блока: <name>" с complexity, confidence, источником оценки,
     reasoning_summary, допущениями и (если есть) идеями оптимизации;
   - в статус-баре правой панели: "Рецензия блока: <name>";
   - приложение не падает; курсор редактора не сдвинулся.
8. Переключить вкладку:
   - кнопки рецензии больше не показываются для предыдущего файла;
   - если в новой вкладке файл с подсвеченными блоками, кнопки появляются
     рядом с подписью сложности для блоков этого файла.
9. Выключить Big-O режим (повторное нажатие на кнопку Big-O в шапке):
   - все decorations и кнопки рецензии исчезают.
10. Включить чекбокс "Использовать ИИ для сложных блоков" в правой панели,
   перезапустить анализ. Для блоков, по которым правил было недостаточно,
   у кнопки в hover-tooltip написано "AI" в подписи; в рецензии блока —
   "Источник оценки: llm (AI)".
11. Если Ollama не запущена, но AI включён: анализ не падает; кнопка
    рецензии работает; в тексте написано "AI-рецензия недоступна".

## Связка JS → Python

- `CustomMonaco._load_editor` подменяет `MonacoPage` на `_BigOReviewPage`
  (подкласс с `acceptNavigationRequest` для URL-схемы `bigo-review:`).
  Канал `_channel` qtmonaco перепривязывается к новой странице, поэтому
  оригинальный `connector` продолжает работать.
- Кнопка создаётся внутри Big-O view zone, рядом с `[O(...)]`; старый hover
  ContentWidget-слой не получает rows и не устанавливается.
- В JS при клике кнопки создаётся невидимый iframe со src
  `bigo-review:/<encoded blockId>`. `acceptNavigationRequest` ловит запрос,
  отменяет навигацию и эмитит `BigOBridge.block_review_requested(str)`.
- `CustomMonaco` пробрасывает сигнал наружу как
  `editor.block_review_requested` (Signal(str)).
- `BigOController.__init__` подключает этот сигнал к `review_block`.

Важно: НЕ вызываем `new QWebChannel(...)` второй раз из JS — это бы
перетёрло `transport.onmessage` qtmonaco и сообщения от Python к Monaco
(включая `set_text`) перестали бы доходить, оставляя редактор пустым.

## Ограничения

- Кнопка находится в view zone рядом с подписью сложности, а не в конце первой
  строки блока; отдельная кнопка создаётся для каждой Big-O подписи.
- Если несколько блоков начинаются на одной строке, view zone остаётся одна
  (по существующей логике `seenZoneLine`) и кнопка будет соответствовать
  первой строке/первому overlay row.
- Рецензия блока временно заменяет текст проектной рецензии в правой
  панели; переключение режима не сделано — для возврата нужно перезапустить
  Big-O анализ или нажать кнопку другого блока.
- `BigOBridge` пока используется только для `reviewBlock`; новые JS↔Python
  каналы (например, "анализировать выделение") можно добавить slot-ами на
  тот же мост, не трогая qtmonaco.
